# 本地开发 — 如何开启与关闭前端 / 后端 / 数据库

本文说明在本机开发时，三个服务分别是什么、如何启动与关闭，以及数据是否会丢失。

---

## 整体架构

本地开发与线上逻辑一致，都是三层结构：**前端 → 后端 API → 数据库**。前端不直接连接数据库。

```
浏览器
   ↓  http://localhost:5173
前端（Vite + React）
   ↓  VITE_API_URL → http://localhost:3001  （见项目根目录 .env.local）
后端（Express，server/）
   ↓  DATABASE_URL → postgres://localhost:5432/fitness  （见 server/.env）
PostgreSQL 数据库（库名 fitness）
```

| 组件 | 默认端口 | 配置文件 |
|------|----------|----------|
| 前端 | **5173** | 根目录 `.env.local` |
| 后端 API | **3001** | `server/.env` |
| PostgreSQL | **5432** | `server/.env` 中的 `DATABASE_URL` |

---

## 重要说明

1. **根目录 `npm run dev` 只启动前端**，不会自动启动后端。
2. **数据库**由 Homebrew 或 Docker 单独管理，不是 `npm run dev` 拉起的。
3. **正常关闭数据库再开启，数据不会丢失**；数据保存在磁盘目录中（见文末「数据存在哪」）。

---

## 一、数据库（PostgreSQL，端口 5432）

本项目使用 **PostgreSQL**（不是 MySQL）。请用 `psql`、TablePlus、DBeaver 等 **PostgreSQL 客户端**查看数据，不要用 MySQL Workbench。

### 方式 A：Homebrew（推荐，当前本机常用）

**首次安装（若尚未安装）：**

```bash
brew install postgresql@16
brew services start postgresql@16
createdb fitness   # 仅第一次需要；若已存在可跳过

# 首次还需执行迁移（仅第一次）
cd "/Users/davidzhan/Desktop/健身APP"
for f in server/migrations/*.sql; do
  /opt/homebrew/opt/postgresql@16/bin/psql -d fitness -f "$f"
done
```

**开启：**

```bash
brew services start postgresql@16
```

**关闭：**

```bash
brew services stop postgresql@16
```

**查看是否在运行：**

```bash
brew services list | grep postgresql
# started = 运行中；stopped = 已关闭
```

**进入数据库命令行：**

```bash
/opt/homebrew/opt/postgresql@16/bin/psql -d fitness
```

常用命令：`\dt` 列出表，`\q` 退出。

### 方式 B：Docker（需已安装 Docker Desktop）

```bash
cd deploy
docker compose up -d postgres    # 仅数据库
# 或
docker compose up -d             # 数据库 + API + Nginx（完整栈）
```

**关闭：**

```bash
cd deploy
docker compose stop postgres     # 仅停数据库
# 或
docker compose down              # 停掉 compose 里所有服务
```

> 若本机未安装 Docker，终端会提示 `command not found: docker`，请改用上面的 Homebrew 方式。

## 二、后端 API（端口 3001）

**前置条件：** 数据库已启动，且已配置 `server/.env`（可复制 `server/.env.example`）。

**开启：**

```bash
cd "/Users/davidzhan/Desktop/健身APP/server"
npm install          # 仅第一次
npm run dev
```

终端出现 `API listening on http://0.0.0.0:3001` 即表示成功。

**关闭：**

在运行后端的终端按 **`Ctrl + C`**。

或在任意终端结束占用 3001 的进程：

```bash
lsof -i :3001
kill <PID>
```

**检查是否正常：**

```bash
curl http://localhost:3001/health
# 应返回 {"ok":true}
```

## 三、前端（端口 5173）

**前置条件：** 根目录已有 `.env.local`，且包含：

```env
VITE_API_URL=http://localhost:3001
```

**开启：**

```bash
cd "/Users/davidzhan/Desktop/健身APP"
npm install          # 仅第一次
npm run dev
```

浏览器访问：**http://localhost:5173**

**关闭：**

在运行前端的终端按 **`Ctrl + C`**。

## 四、日常开发推荐流程

### 开始开发（开三个东西）

建议开 **两个终端** + 确认数据库在跑：

| 顺序 | 操作 | 终端 |
|------|------|------|
| 1 | `brew services start postgresql@16` | 任意（若已是 started 可跳过） |
| 2 | `cd server && npm run dev` | 终端 1 — 后端 |
| 3 | `cd 项目根目录 && npm run dev` | 终端 2 — 前端 |

### 结束开发（关三个东西）

| 顺序 | 操作 |
|------|------|
| 1 | 前端终端 **`Ctrl + C`** |
| 2 | 后端终端 **`Ctrl + C`** |
| 3 | （可选）`brew services stop postgresql@16` — 想省内存时再关库 |

不开发时关掉后端和数据库，可减少本机内存占用；下次 `start` 后数据仍在。

## 五、快速对照表

| 组件 | 开启 | 关闭 | 端口 |
|------|------|------|------|
| 数据库 | `brew services start postgresql@16` | `brew services stop postgresql@16` | 5432 |
| 后端 | `cd server && npm run dev` | 该终端 `Ctrl + C` | 3001 |
| 前端 | `npm run dev`（项目根目录） | 该终端 `Ctrl + C` | 5173 |

## 六、数据会丢吗？

| 操作 | 数据 |
|------|------|
| 只关前端 / 后端 | **不丢** |
| `brew services stop` 后再 `start` 数据库 | **不丢** |
| 重启电脑后再 `start` 数据库 | **不丢** |
| `dropdb fitness` 后重建 | **会清空** fitness 库 |
| 删除 `/opt/homebrew/var/postgresql@16` | **会清空** 本机 PostgreSQL 全部数据 |

---

## 七、数据存在哪（Homebrew）

PostgreSQL 数据文件目录（一般无需手动改）：

```text
/opt/homebrew/var/postgresql@16
```

库名 **`fitness`** 内的表包括：`users`、`profiles`、`day_logs`、`exercises`、`meals`、`follows`、`day_likes`、`day_comments` 等。

## 八、内存占用（简要）

| 组件 | 不开发时建议 |
|------|----------------|
| 前端 Vite | 关掉终端即释放 |
| 后端 Node | `Ctrl + C` 关闭 |
| PostgreSQL | 不用时可 `brew services stop`，释放几十 MB～百余 MB |

## 九、常见问题

### 页面提示「无法连接服务器，请检查网络后重试」

- 后端未启动：在 `server` 目录执行 `npm run dev`。
- 或 `curl http://localhost:3001/health` 失败，先修好 API 再开前端。

### 后端启动报错，连不上数据库

- 确认数据库已 `brew services start postgresql@16`。
- 确认 `server/.env` 中 `DATABASE_URL` 正确，例如：  
  `postgres://localhost:5432/fitness`

### MySQL Workbench 连不上 fitness

- 本项目是 **PostgreSQL**，请改用 TablePlus、DBeaver（选 PostgreSQL）或 `psql`。

### 端口已被占用

```bash
lsof -i :3001   # 后端
lsof -i :5173   # 前端
lsof -i :5432   # 数据库
```
