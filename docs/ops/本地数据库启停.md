# 本地数据库启停（localhost）

> 索引：[ops/README.md](README.md) · [开发入门](../GETTING-START.md)

本地开发时，数据库使用 **Homebrew 安装的 PostgreSQL 16**，在本机 `5432` 端口运行。  
**不需要 Docker**，也**不用**进 `deploy` 目录（`deploy` 仅用于腾讯云服务器部署）。

前端、后端仍在项目里用 `npm run dev` 启动；只有数据库由 Homebrew 管理。

---

## 三个端口分别是什么

| 端口 | 服务 | 怎么开 |
|------|------|--------|
| **5432** | PostgreSQL 数据库 | `brew services start postgresql@16` |
| **3001** | 后端 API | `cd server && npm run dev` |
| **5173** | 前端页面 | 项目根目录 `npm run dev` |

数据流：`浏览器 → 5173 前端 → 3001 后端 → 5432 数据库`（前端不直连数据库）。

---

## 前提

- macOS 已安装 [Homebrew](https://brew.sh/)
- 已安装 PostgreSQL 16（见下方「首次安装」）

---

## 首次安装（只做一次）

### 1. 安装 PostgreSQL

```bash
brew install postgresql@16
```

### 2. 把命令行工具加入 PATH（建议）

Apple 芯片 Mac 在 `~/.zshrc` 末尾加一行：

```bash
export PATH="/opt/homebrew/opt/postgresql@16/bin:$PATH"
```

Intel Mac 改为：

```bash
export PATH="/usr/local/opt/postgresql@16/bin:$PATH"
```

保存后执行：

```bash
source ~/.zshrc
```

### 3. 启动服务并创建数据库

```bash
brew services start postgresql@16
createdb fitness
```

### 4. 执行迁移（建表）

在项目根目录执行（按文件名顺序）：

```bash
for f in server/migrations/*.sql; do
  echo "==> $f"
  psql -d fitness -f "$f"
done
```

若某条迁移报错「已存在」，多半表示之前跑过，可忽略或只补跑新的 `.sql` 文件。

### 5. 配置后端环境变量

```bash
cp server/.env.example server/.env
```

本地 Homebrew 默认**无密码**，`server/.env` 中建议为：

```text
DATABASE_URL=postgres://localhost:5432/fitness
```

---

## 开启数据库

```bash
brew services start postgresql@16
```

**查看是否在运行：**

```bash
brew services list | grep postgresql
```

`postgresql@16` 一行显示 **started** 即表示已开启。

**或用端口检查：**

```bash
lsof -i :5432
```

有 `postgres` 进程即正常。

---

## 关闭数据库

```bash
brew services stop postgresql@16
```

再次 `brew services list`，状态应为 **stopped**。

### 关库再开，数据会丢吗？

**不会。** 打卡、账号等都在磁盘上，只有执行「删库」才会清空。

| 操作 | 数据 |
|------|------|
| `brew services stop` 后再 `start` | 保留 |
| 重启 Mac 后再 `start` | 保留 |
| `dropdb fitness` | **清空该库** |

数据目录大致在：

- Apple 芯片：`/opt/homebrew/var/postgresql@16`
- Intel：`/usr/local/var/postgresql@16`

## 查看本地数据库

### 命令行

```bash
psql -d fitness
```

常用命令：

- `\dt` — 列出所有表  
- `\d users` — 查看某表结构  
- `select email from users;` — 查询  
- `\q` — 退出  

### 图形工具

请用 **PostgreSQL** 客户端（如 TablePlus、DBeaver、pgAdmin），**不要用 MySQL Workbench**（协议不同）。

连接参数示例：

| 项 | 值 |
|----|-----|
| Host | `localhost` |
| Port | `5432` |
| Database | `fitness` |
| User | 一般为你的 Mac 登录名（Homebrew 默认） |
| Password | 通常留空 |

## 删除本地数据库（慎用）

只删**数据**，不卸载 PostgreSQL：

```bash
brew services stop postgresql@16   # 可选，停服再删更稳妥
dropdb fitness
```

若要**重新建空库**：

```bash
brew services start postgresql@16
createdb fitness
for f in server/migrations/*.sql; do psql -d fitness -f "$f"; done
```

之后需在 App 里**重新注册账号**。

## 日常开发推荐流程

**开始：**

```bash
# 1. 数据库（若已是 started，可跳过）
brew services start postgresql@16

# 2. 后端（终端 1）
cd server && npm run dev

# 3. 前端（终端 2，项目根目录）
npm run dev
```

浏览器打开：`http://localhost:5173`  
确认根目录 `.env.local` 中有：`VITE_API_URL=http://localhost:3001`

**结束：**

1. 在前端、后端终端按 `Ctrl + C` 停掉 5173、3001  
2. 若希望释放内存，再执行：`brew services stop postgresql@16`  
   （不关库也可以，下次开发省一步）

## 常见问题

### 1. `command not found: psql` / `createdb`

未把 PostgreSQL 加入 PATH，见上文「首次安装」第 2 步，或临时使用完整路径：

```bash
/opt/homebrew/opt/postgresql@16/bin/psql -d fitness
```

### 2. `createdb: database "fitness" already exists`

库已存在，无需再建，直接 `psql -d fitness` 即可。

### 3. API 连不上数据库 / `ECONNREFUSED 5432`

先执行 `brew services start postgresql@16`，再重启 `cd server && npm run dev`。

### 4. `brew services start` 失败或端口被占用

查看谁占用 5432：

```bash
lsof -i :5432
```

若装了多个 PostgreSQL，只保留一个版本在跑，或改掉冲突服务。

### 5. 与腾讯云的关系

- **本地**：Homebrew + `server/.env`  
- **服务器**：`deploy/` 目录下的 Docker（与本地无关）

本地改代码、连本地库，**不需要**在本机执行 `docker compose`。
