# 腾讯云部署 — 一步一步做（前端 + 后端）

> 索引：[ops/README.md](README.md) · [部署说明](../architecture/deploy.md)

按顺序完成每一阶段，做完一项打勾 ✓。你的手机最终打开的是：**http://服务器公网IP**（或以后的域名）。

---

## 整体架构（你会得到什么）

```
家人手机浏览器
    ↓  http://你的公网IP
腾讯云轻量服务器 (Ubuntu)
    ├── 80 端口 Nginx  →  前端网页 (dist/)
    └── /api/*         →  后端 Node (server/)
                              └── PostgreSQL (Docker 数据库)
```

- **前端**：打包后的静态网页，由 Nginx 托管  
- **后端**：`server/` API + 数据库，家人注册登录、记运动饮食  
- **不需要 Supabase**，**不需要邮箱验证**

## 阶段 0：准备（在你 Mac 上）

```bash

npm run build
```

最后应显示 `✓ built`，无红色报错。

### 0.2 准备记录这些信息（后面会填）

| 名称 | 你填的值 |
|------|----------|
| 服务器公网 IP | 例：`123.45.67.89` |
| SSH 登录用户 | 一般是 `root` |

| JWT 密钥（自拟） | 随便一长串，至少 32 字符 |

## 阶段 1：购买腾讯云轻量服务器（约 10 分钟）

1. 打开 [腾讯云轻量应用服务器](https://cloud.tencent.com/product/lighthouse) 并登录  
2. **新建** → 配置建议：  
   - 地域：上海 / 广州（离你家近）  
   - 镜像：**Ubuntu 22.04 LTS**  
   - 套餐：**2核 2GB**（家庭够用）  
3. 设置 **root 密码**（务必保存）  
4. 购买完成后，在控制台找到该实例的 **公网 IP**

### 1.1 放行防火墙端口

在轻量控制台 → 该实例 → **防火墙**（或安全组）→ 添加规则：

| 端口 | 说明 |
|------|------|
| 22 | SSH 登录 |
| 80 | 网站 + API（必须） |
| 443 | 以后上 HTTPS 可选 |

✓ 阶段 1 完成：你能看到公网 IP，且 22、80 已放行。

---

## 阶段 2：SSH 登录并安装 Docker（约 15 分钟）

### 2.1 在本机终端连接服务器

把 `你的公网IP` 换成真实 IP：

```bash
ssh root@你的公网IP
```

第一次会问 `yes/no`，输入 `yes`，再输入 root 密码。

### 2.2 在服务器上安装 Docker

连接成功后，**在服务器里**执行。

**若是 Ubuntu**（`apt` 可用）：

```bash
curl -fsSL https://get.docker.com | sh
systemctl enable --now docker
apt update && apt install -y docker-compose-plugin git
```

**若是 OpenCloudOS / CentOS**（`apt` 不存在，不要用 get.docker.com）：

```bash
dnf install -y docker docker-compose-plugin git
systemctl enable --now docker
```

若提示找不到 `docker-compose-plugin`，改用 Docker 官方源：

```bash
dnf install -y dnf-plugins-core
dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
dnf install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin git
systemctl enable --now docker
```

验证：

```bash
docker --version
docker compose version
```

✓ 阶段 2 完成：你能 `ssh` 进服务器，且 `docker compose version` 有输出。

## 阶段 3：把项目放到服务器（二选一）

### 方式 A — 用 Git（推荐，若代码在 GitHub）

**在服务器上：**

```bash
mkdir -p /opt/fitness-app
cd /opt
git clone https://github.com/你的用户名/你的仓库.git fitness-app
cd fitness-app
```

### 方式 B — 用 scp 从 Mac 上传（没有 GitHub 时）

**先在你 Mac 上**打包（排除 node_modules 减小体积）：

```bash
cd "/Users/davidzhan/Desktop"
tar czf fitness-app.tar.gz --exclude=健身APP/node_modules --exclude=健身APP/server/node_modules 健身APP
scp fitness-app.tar.gz root@你的公网IP:/opt/
```

**再在服务器上：**

```bash
cd /opt
tar xzf fitness-app.tar.gz
mv 健身APP fitness-app
cd fitness-app
```

✓ 阶段 3 完成：服务器上存在 `/opt/fitness-app`，且有 `deploy/`、`server/` 目录。

## 阶段 4：在 Mac 上构建「指向服务器」的前端（约 5 分钟）

**在你 Mac 上**（不要 SSH 进服务器做这步）：

```bash
cd "/Users/davidzhan/Desktop/健身APP"
SERVER_IP=你的公网IP ./scripts/build-for-tencent.sh
```

例如：

```bash
SERVER_IP=123.45.67.89 ./scripts/build-for-tencent.sh
```

成功后会生成 `dist/` 文件夹（里面是前端网页）。

### 若用方式 B（scp）上传的，需要再传一次 dist

```bash
scp -r dist root@你的公网IP:/opt/fitness-app/
```

若用 Git，在服务器上 `git pull` 后也要在 **Mac 上 build 完再把 dist 传上去**，或见下方「在服务器构建」备选。

✓ 阶段 4 完成：服务器 `/opt/fitness-app/dist/` 里有 `index.html` 等文件。

## 阶段 5：配置服务器环境变量（约 5 分钟）

**SSH 进服务器：**

```bash
cd /opt/fitness-app/deploy
cp .env.example .env
vi .env
```

把内容改成类似（**换成你的真实值**）：

```env
POSTGRES_PASSWORD=这里写你的强密码16位以上
JWT_SECRET=这里写至少32位随机字符串
CORS_ORIGIN=http://你的公网IP
DEEPSEEK_API_KEY=sk-你的DeepSeek密钥
```

保存：`Esc` 后输入 `:wq` 回车（`vi` 编辑器；若无 `nano` 可用 `dnf install -y nano`）。

AI 估算原理与排错详见 [服务器配置-DeepSeek与API密钥.md](./服务器配置-DeepSeek与API密钥.md)。

✓ 阶段 5 完成：`deploy/.env` 已保存且密码不是默认的 `change_me`。

## 阶段 6：启动 Docker（前端 + 后端 + 数据库）（约 10 分钟）

仍在服务器 `/opt/fitness-app/deploy`：

```bash
cd /opt/fitness-app/deploy
docker compose up -d --build
```

第一次会下载镜像，需几分钟。完成后：

```bash
docker compose ps
```

应看到 `postgres`、`api`、`web` 三个服务都是 `running`。

### 6.1 自检

```bash
curl -s http://127.0.0.1/health
curl -s http://127.0.0.1/api/health
```

都应返回 `{"ok":true}` 或类似 JSON。

✓ 阶段 6 完成：两条 curl 都成功。

## 阶段 7：用手机 / 浏览器访问（验收）

1. 手机连 WiFi 或 4G（不要用仅内网的环境）  
2. 浏览器打开：**http://你的公网IP**

- **iPhone Safari**：分享 → 添加到主屏幕  
- **Android Chrome**：菜单 → 安装应用 / 添加到主屏幕  

✓ 阶段 7 完成：家人各自注册账号即可长期使用。

## 阶段 8：以后更新 App 时怎么做

**首次配置（只需一次）：**

```bash
cd "/Users/davidzhan/Desktop/健身APP"
cp .env.deploy.example .env.deploy
# 编辑 .env.deploy，填写 SERVER_IP=211.159.166.109
```

**日常更新（在 Mac 上）：**

```bash
npm run deploy:tencent        # 只改了前端 → 自动构建 + 上传 dist
npm run deploy:tencent:api    # 也改了 server/ → 额外重建 api 容器
```

执行时会提示输入 SSH 密码（与登录服务器相同）。

## 常见问题

### 打不开网页 / 一直转圈

- 检查防火墙是否放行 **80**  
- 服务器上：`docker compose ps` 是否三个都在运行  
- `curl http://127.0.0.1` 在服务器上是否能看到 HTML  

### 能打开网页但注册失败

- 浏览器 F12 → Network 看请求是否发到 `http://你的IP/api/...`  
- 检查构建时是否用了 `SERVER_IP=正确IP ./scripts/build-for-tencent.sh`  
- 检查 `deploy/.env` 里 `CORS_ORIGIN=http://你的IP`（不要末尾斜杠）

### dist 目录不存在导致 web 起不来

必须先阶段 4 构建，再 `docker compose up`。  
或：`docker compose up -d postgres api` 先起后端，补传 `dist` 后 `docker compose up -d web`。

### 想绑定域名 + HTTPS

需域名 **备案**（国内服务器）。备案后申请 SSL 证书，改 Nginx 443，并把构建时的地址改为 `https://域名/api`。可后续再做。
